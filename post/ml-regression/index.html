机器学习中的线性回归

Oct 6, 2016

模型

hypothesis

$$ h_{\theta}= \theta X = {\sum}_{i=1}^{n} x_i’ \theta $$

其中：

X is a m x n matrix
n is the number of the features
m is the number of training examples.

gradient

\[ \theta_j :=\theta_j- \alpha \frac{\partial}{\partial \theta_j} J(\theta_j) \] 写成矩阵形式即： \[ \theta := \theta - \alpha X^T(X \theta - y) \]

cost function:

\[ J(\theta) = \frac12 {(X \theta - y)}^T (X \theta - y) = \frac12 \sum_{i=1}^m(h_\theta(x^{(i)}) - y^{(i)})^2 \]

解法

梯度下降法

核心思想：选择初始点，不断迭代直到找到最优点。

为什么取 -$\frac{\partial}{\partial \theta_j} J(\theta_j)$为方向？为什么不是正的，或者是正负之间的某个夹角？

根据 泰勒展开式 \[ J(x + \alpha) \approx J(x) + \alpha \nabla J(x) = J(x) + | \alpha | |\nabla | \cos \theta \] 其中如果选择同方向，会使得 $\cos \theta = 1 $,反方向使得 $\cos \theta = -1$

因此选择反方向能够使得 $J(\theta) $最快下降。

batch gradient descent

gradient:

\[ \theta_j := \theta_j - \alpha \frac{\partial}{\partial \theta_j} J(\theta) \]

\[ \theta := \theta - \alpha X^T(X\theta - y) \]

$\theta_j $的每一步都是迭代都是将训练集中的所有样本点都遍历一遍才走出的，这样在大样本情况下计算量很大，这样也就有了下面的 Stochastic gradient descent

stocstic gradient descent

每一个样本点都会走微小的一步，这样不需要遍历$m$项才走了。 \[ cost(\theta, (x^{(i), y^{(i)}})) = 1⁄2 (h_{\theta}(x^{(i)}) - y^{(i)})^2 \]

\[ J_{(train)}(\theta ) = 1/m \sum_{m = 1}^m cost(\theta, (X^{(i)}, y^{(i)})) \]

具体步骤：

首先需要把数据集随机排列，然后：

Repeat { for i = 1,…,m{ \[ \theta_j := \theta_j - \alpha (h_{\theta} (x^i) - y^i) x^i_j for j = 0,…,n \] }

}

依靠样本点的挨个输入来调参，会比 batch方法走得更加曲折，最后也不会停在全局最小点（如果能够找到的话），而是在最小附近打转。

经验表明外部的repeat，即从 $1 $到 $m $完全执行完成，需要大概 $1-10$ 圈能够找到全局最小点（如果能够找到的话）

而使用 batch gradient descent的方法，$m $极大，需要计算很久才能够走上一步。

mini-Batch gradient descent

$eg: b= 10, examples((x^i, y^i), …, (x^{i+9}, y^{i+9}))$

\[ \theta_j := \theta_j - \alpha (1⁄10) \sum_{k=i}^{i+9} (h_{\theta}(x^{k}) - y^k) x^j_k \]

\[ i := i+10 \]

伪代码(b = 10, m = 1000)

repeat{ for i = 1, 11, 21, 31, 991{ \[ \theta_j = \theta_j - \alpha (1⁄10) \sum_{k=i}^{i+9} (h_{\theta}(x^{k}) - y^k) x^j_k \ for \ every \ j = 0,…,n \] } }

各种方法的对比

batch: 每次循环使用所有 m个样本点

sotchastic: 每次循环只用 1 个样本点

mini-batch: 使用 b 个样本点每次循环（b: mini-batch size, 可取 b = 2:100
