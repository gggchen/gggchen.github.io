Emacs cannot find pdflatex PATH

Nov 20, 2017

Problem description

After installing Spacemacs, my org file cannot export to PDF file, the following error message always pops out:

command not found: pdflatex

Solution

It finally turns out it is my Macbook move /usr/texbin to /Library/TeX/texbin, according to this answer, I added export PATH="$PATH:/Library/TeX/texbin"to my .zshrc. and then

source .zshrc

to update my settings.

After restarting my emacs, finally Emacs can export org file to PDF file. :D

Rule out other possible problem

I also tried to solve this by some other methods but failed.

Possibility1: no Latex installed in my computer

But I can build PDF from .tex file smoothly in Sublime. It seems that Sublime could find pdflatex in my computer but Emacs cannot.

Possibility2: Emacs did not include my system zsh PATH thus fail to find pdflatex

When I run which pdflatex, zsh cannot find my pdflatex as well.

which pdflatex
pdflatex not found

There is no pdflatex recognized by my zsh as well. And What I use is Spacemacs thus exec-path-from-shell is already installed for me to ensure environment variables inside Emacs look the same as in the user’s shell.

It turns out there is no texbin under my /usr directory.

I Google this problem. Mac OS X El Capitan moved my /usr/texbin to /Library/TeX/texbin.
