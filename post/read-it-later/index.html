稍后读并且存档

Jul 14, 2017

解决的问题:

  1. 长文章当时读不完就收藏了, 但是收藏的文章从来不会读完
  2. 读过的文章没有将整个网页保存起来, 以后网页一旦失效, 就找不到了

解决方法:

  1. 使用 Pocket 或者 Instapaper
  2. 使用 ifttt 将 Pocket 读完的文章自动存档到 Evernote

稍后读

之前在 Safari Reading List, Evernote 等地方收藏了一堆的文章, 但是我从来不读. 就像一直戏谑的, read it later 变成 read it never. 后来听了 Checked 某一期的博客, 认识到了而且可以用 Pocket 或者 Instapaper 解决, 就开始使用了这个 Pocket 作为稍后读的工具.

我的稍后读流程

  1. 浏览 RSS 订阅的消息. 我使用 Reeder3 来订阅 RSS 源, 如果觉得读某一篇时觉得一时没有时间读, 就用 Reeder3 发送到 Pocket
  2. 平时各种空闲时间, 不打开微信各种社交软件, 而是打开 Pocket 查看之前收藏的文章, 这样一天大概是能够把之前加入的文章读完的
  3. 在 Pocket 读完的文章, 勾选读完之后, ifttt 会自动帮我把文章全文存档到 Evernote

下面详细说明利用 ifttt 存档到 Evernote

笔记存档

需要做的准备:

  • 一个任意的 Gmail 邮箱
  • Evernote 的中级或高级帐号. Evernote 需要购买中级或者高级版本, 才可以使用发送邮件到 Evernote 的邮箱保存该邮件为笔记的功能.
  • Evernote 在settings/account summary/Email notes to 下面, 找到形式为 xxxx.xxxx@m.evernote.com 的邮箱, 这是用来接受笔记并且保存的邮箱
  • 安装并且注册 ifttt

配置方法

Ukeeper

在 http://www.ukeeper.com/ 用邮箱注册账户, 把用来发消息的 Gmail 邮箱填写到 Regular e-mail, Evernote 的邮箱填写到 Forwarding e-mail.

ifttt

在 ifttt 中使用我所创建的一个 applet

绑定 Gmail 邮箱(和上面 ukeeper 的 Regular e-mail 一致), 绑定 Pocket 账户, 就能愉快地使用了. 这里的 To Address 中的邮箱选择默认的 drops@ukeeper.com 就好, 不需要做更改.

ifttt 的 applet 配置原理

  • 如果 ifttt 检测到我的 Pocket 中 读完(Archive) 了文章
  • 那么 ifttt 就会利用绑定的 Gmail 发送一封邮件到 drops@ukeeper.com, 标题为”文章的标题 + @Archive, #Pocket, #Tags”, 邮件正文为 Pocket 保存文章的 Url
  • 然后 ukeeper.com 会自动把 Url 提取出页面, 变成邮件正文, 而标题仍然是 “文章的标题 + @Archive, #Pocket, #Tags”(@Archive 为指定 Evernote 中保存文件夹到 Archive 文件夹, 标签为 pocket 和文章的其他标签) 发送到我的notebook 的 Archive 文件夹, 贴上 pocket 和 Pocket 原有的标签

这里之所以执着于标签, 是想像 Checked 的主播 Hum 一样, 想让 Pocket 和 Evernote 使用同一套标签体系. 而之所以用标签, 而不是文件夹管理, 是因为文件夹的层次结构树状的, 就是一篇文章在A目录下, 就不能在 B 目录下. 而不同标签可以加在同一文件上.

Debug

如果并不能实现 Pocket 发送文章保存到 Evernote 的功能, 请测试:

  1. ifttt 是否工作: 将 ifttt 中的 To Address 改为你的任意其他邮箱, 在 Pocket 中 Archive 一篇文章, 是否在邮箱中能收到由 ifttt 发送到该邮箱的邮件.
  2. ukeeper 和 Evernote 是否工作, 用注册 ukeeper 的 Gmail 邮箱向 drops@ukeeper.com 发送一封邮件, 是否在 Evernote 中能够收到保存的笔记

下面附上我 ifttt filter code 的代码:

var PocketTagList = Pocket.newReadItemRil.Tags.split(', ');
var PocketTag = PocketTagList.join(', #');

Gmail.sendAnEmail.setSubject(Pocket.newReadItemRil.Title + ' @Archive, #Pocket, #' + PocketTag);
Gmail.sendAnEmail.setBody(Pocket.newReadItemRil.Url);
